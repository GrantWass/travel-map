# TRAVEL APP DATABASE STRUCTURE

## Core Idea

The database revolves around USERS (travelers) and TRIPS.

Users can:

* create trips
* comment on trips
* friend other users
* favorite users or trips
* plan trips
* save activities and lodgings

Trips contain:

* activities
* lodgings
* tags
* comments

---

## TABLE: travelers

Primary user table.

Columns:

* user_id (PK)
* cognito_sub (UUID from auth system)
* name
* email
* bio
* verified (boolean)
* college
* profile_image_url
* password_hash
* saved_activity_ids (array of activity ids)
* saved_lodging_ids (array of lodging ids)
* completed_onboarding_tours (json)
* phone

Relationships:

* owns trips
* writes comments
* sends SMS invites
* friends other users
* favorites users
* favorites trips
* plans trips

---

## TABLE: trips

Trips created by users.

Columns:

* trip_id (PK)
* thumbnail_url
* title
* description
* latitude
* longitude
* cost
* duration
* date
* visibility
* owner_user_id (FK → travelers.user_id)
* event_start
* event_end
* geo_location (PostGIS geographic object)

Relationships:

* belongs to traveler (owner_user_id)
* contains activities
* contains lodgings
* contains comments
* contains tags
* can be favorited by users
* can be planned by users

---

## TABLE: activities

Things to do during a trip.

Columns:

* activity_id (PK)
* trip_id (FK → trips.trip_id)
* address
* thumbnail_url
* title
* location
* description
* latitude
* longitude
* cost
* geo_location (PostGIS)

Relationships:

* many activities belong to one trip
* activities may be saved by users

---

## TABLE: lodgings

Places to stay during a trip.

Columns:

* lodge_id (PK)
* trip_id (FK → trips.trip_id)
* address
* thumbnail_url
* title
* description
* latitude
* longitude
* cost
* geo_location (PostGIS)

Relationships:

* many lodgings belong to one trip
* lodgings may be saved by users

---

## TABLE: comments

User comments on trips.

Columns:

* comment_id (PK)
* user_id (FK → travelers.user_id)
* trip_id (FK → trips.trip_id)
* body
* created_at

Relationships:

* comment belongs to a user
* comment belongs to a trip

---

## TABLE: friendships

Stores friend relationships between users.

Columns:

* id (PK)
* requester_id (FK → travelers.user_id)
* addressee_id (FK → travelers.user_id)
* status (pending / accepted / declined)

Relationships:

* user ↔ user

---

## TABLE: sms_invites

Tracks SMS invites sent by users.

Columns:

* id (PK)
* inviter_id (FK → travelers.user_id)
* phone_number
* invite_token
* status
* claimed_user_id (FK → travelers.user_id)
* created_at

Relationships:

* user sends invite
* invite may become a user

---

## TABLE: trip_tags

Tags attached to trips.

Columns:

* trip_id (FK → trips.trip_id)
* tag

Relationships:

* many tags per trip

---

## TABLE: user_favorite_travelers

Tracks users favoriting other users.

Columns:

* user_id
* favorited_user_id

Relationships:

* many-to-many users

---

## TABLE: user_favorite_trips

Tracks users favoriting trips.

Columns:

* user_id
* trip_id

Relationships:

* many-to-many users ↔ trips

---

## TABLE: user_planned_trips

Trips a user plans to take.

Columns:

* user_id
* trip_id

Relationships:

* many-to-many users ↔ trips

---

## TABLE: user_comment_read_state

Tracks comment-notification read cursor for each user.

Columns:

* user_id (PK, FK → travelers.user_id)
* last_seen_trip_comment_at (timestamp with timezone)

Relationships:

* one row per user
* used to calculate unread comments on trips they own

---

## COMMENT NOTIFICATION API STRUCTURES

### GET /users/me/notifications/comments/unread-count

Response:

* unread_count (integer total)
* unread_count_by_trip (dictionary/map of trip_id → count)

Example:

* unread_count = 7
* unread_count_by_trip = {"12": 2, "44": 5}

### POST /users/me/notifications/comments/mark-read

Response:

* message
* last_seen_at (ISO timestamp)

│
├── friendships (user ↔ user)
│
├── favorites
│       ├── user_favorite_trips

│
├── planning
│       └── user_planned_trips
│

---

## MENTAL MODEL

Users create trips.

Trips contain:

* activities
* lodgings
* comments
* tags

Users interact with trips by:

* commenting
* favoriting
* planning

Users interact with other users by:

* friendships
* favorites
* SMS invites
